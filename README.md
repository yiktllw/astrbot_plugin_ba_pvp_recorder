# 蔚蓝档案竞技场战报记录

## 声明

本项目由 AI 生成，功能可能存在缺陷。

## 功能

Astrbot 的蔚蓝档案竞技场战报记录插件。

自动识别指定群聊中的图片，若为竞技场战报，则调用 LLM 识别战报信息。

**不会**自动识别转发消息中的图片，此设计是为了防止转发大量不相干的图片时消耗过多 token。

**请使用带有视觉理解功能的模型**。

## 数据更新

插件主进程会在每天 18:00 自动执行一次数据更新，数据源为 SchaleDB。

## 命令

- `/帮助`

  查看帮助。
- `/记录`

  手动记录战报，有如下用法：
  - `/记录 [图片][图片]`
  - `[引用图片] /记录`
  - `[引用转发消息] /记录`
- `/查询战报 [用户名] [YYYY-MM-DD(可选)]`：

  按人/按天查询战报。
- `/最近战报 [用户名(可选)] [条数(可选)]`

  查询最近记录的战报，默认查询 5 条。
- `/今日战报 [用户名(可选)]`

  查询今日战报，以凌晨 3 点为日期分界线。
- `/队伍`

  按队伍阵容查询战报。可以用常见简称及类型代替角色：
  - 角色简称：如`花`代表水花，`跑`代表鹤城，`白`代表水白子，`s`代表瞬。
  - t/c：t 代表坦克，c 代表输出。
  - 数字 3 到 8：代表角色射程。比如 4 代表射程 450。
  - 下划线`_`用来指代任何学生。

  有如下用法：
  - `/队伍 [单字简称字符串]`

    每个字符识别为一个角色，总共 4 到 6 字。比如：
    - `/队伍 ttt花`
    - `/队伍 花t跑t响`
    - `/队伍 tt4花白咲`
    - `/队伍 tt_t`
  
    *如果某个学生的简称有两字及以上 (如魔芋) ，请使用下面的写法*。
  
  - `/队伍 [st1] [st2] [st3] [st4] [sp1?] [sp2?] [n?]`
  
    与前一用法类似，但是使用空格来分隔学生。

    前四个参数代表前锋学生，第五、六个参数(可选)代表支援学生。第七个参数(可选)代表最大条数。例如：
    - `/队伍 t 花 t s`
    - `/队伍 s t t 7 魔芋`
    - `/队伍 3 3 花 4 _ _ 5`
- `/清空战报 [玩家名称(可选)]`

  清空当前会话的战报。若带有玩家名称，则只清空指定玩家名称的战报。
  
  **该命令会直接删除对应数据，建议配置仅管理员可用**。
